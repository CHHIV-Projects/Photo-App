# Milestone 12.62.7 Closeout — Guided iCloud Source Intake Handoff

## 1. Scope Completed

Implemented a frontend-first guided Source Intake handoff for iCloud Source Profiles inside the Ingestion details drawer.

Completed items:
- Added iCloud-specific Source Intake handoff section in the iCloud readiness/acquisition area.
- Reused existing Source Intake run flow (confirmation modal, payload, validation, polling, terminal summary, stop behavior).
- Added Total Limit suggestion fallback chain from existing data:
  - latest acquisition file_inventory_count when available
  - latest acquisition recent_count when file_inventory_count is unavailable
  - blank/unlimited when neither is available
- Added warning-compatible behavior when staged inventory count is unavailable.
- Preserved manual workflow boundaries (no auto-run orchestration, no cleanup execution).
- Added legacy iCloud profile detection fallback so older cloud_export records without explicit cloud_provider still surface the iCloud handoff.

## 2. Operational Behavior

Guided handoff behavior:
- The handoff appears in the iCloud details area in Ingestion.
- Action is prepared manually by the operator before starting Source Intake.
- Source Profile source_id is used as ingestion_source_id for Source Intake requests.
- Confirmation dialog remains the shared Source Intake dialog, with iCloud-context copy and guidance.

Readiness/guardrail gating:
- Disabled when readiness is not ready, blocking reasons exist, path/source alignment is mismatched, approved root is blocked, or operation conflicts exist.
- Disabled for inactive profiles.
- Allows proceeding with warning when no staged inventory count is available but readiness/path checks are otherwise valid.

Status and terminal behavior:
- Reuses existing Source Intake status polling and terminal summary behavior in Ingestion.
- Reuses existing Request Stop behavior.

Reconnaissance findings captured for this milestone:
- Current Source Intake API helper: startSourceIntake in frontend API layer.
- Current Source Intake payload shape:
  - ingestion_source_id
  - source_intake_limit
  - ingest_batch_size
- Current Source Intake status polling behavior: existing Ingestion polling loop reused.
- Current Source Intake stop behavior: existing stop endpoint and UI flow reused.
- Local/external Run Intake in Ingestion: unchanged and still uses shared run confirmation and start handler.
- iCloud acquisition summary inventory signal: file_inventory_count and recent_count are available from existing acquisition status snapshot.
- ingestion_source_id resolution: Source Profile source_id is used as ingestion_source_id in current flow.
- Staged inventory count availability: no dedicated live staged-folder inventory count exposed without backend changes in this milestone.
