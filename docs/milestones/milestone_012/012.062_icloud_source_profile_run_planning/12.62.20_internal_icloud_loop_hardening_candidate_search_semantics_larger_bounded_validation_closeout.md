## Implemented

- Reclassified the internal loop's old `scan-limit` meaning as `candidate_search_cap`.
- Added `--candidate-search-cap` to the internal operator script.
- Kept `--scan-limit` as a backward-compatible alias only.
- Added report fields clarifying the terminology:
  - `candidate_search_cap`
  - `scan_limit_meaning: candidate_search_cap`
  - `candidate_page_size: null`
  - `candidate_page_size_status: deferred`
  - `candidate_pages_scanned: null`
- Deferred page-size semantics explicitly because the current adapter does not provide real page-by-page acquisition pagination.
- Hardened `ordinary_still_only` planning so non-still candidates, including MOV resources, are counted/reported as unsupported but are not selected for download.
- Preserved the existing safe still behavior:
  - selected ordinary stills must be single-resource primary originals
  - Live Photos, multi-resource assets, sidecars, RAW pairs, ambiguous identities, and selected unsupported resources remain blocked
  - adjustment-metadata-only observations remain non-blocking because the parser removes that non-blocking reason before planning
- Added partial-batch protection:
  - if a batch target is not fully filled, the loop reports `partial_batch_selected`
  - no partial batch is acquired by default
  - `operator_decision_required` is set for partial-batch cases
- Added clearer batch/run report counters:
  - per-batch candidates considered
  - per-batch resource candidates considered
  - unsupported/blocked candidate count
  - safe-but-not-selected count
  - aggregate run candidate/resource candidate counts
- Added final reconciliation/reporting fields:
  - acquisition run IDs
  - acquisition batch IDs
  - Source Intake run IDs
  - ingestion run IDs
  - cleanup dry-run IDs
  - cleanup execution IDs
  - final cleanup verification IDs
  - staging/drop-zone/partial-workspace cleanliness flags
- Added a generated `continue_cleanup_command` in paused-for-cleanup script output.
- Preserved explicit local cleanup confirmation:

```text
DELETE LOCAL STAGING COPIES
```

1. acquire exactly two ordinary still logical items

8. pause for guarded local cleanup review again
9. require explicit cleanup confirmation again
10. complete with `total_limit_reached`
