# 12.62.19 Internal iCloud Multi-Batch Loop Orchestration Closeout

## Status

Implemented and unit-validated.

This milestone adds an internal, non-UI orchestration layer for the exact-selection iCloud workflow:

```text
preflight
-> exact-selection acquisition
-> batch Source Intake
-> guarded cleanup dry run
-> explicit operator cleanup continuation
-> staging-clean verification
-> repeat until total limit / no more items / blocked / failed
```

No normal UI route, normal Admin API route, cloud deletion behavior, or production one-click flow was added.

## Implemented

- Added lightweight internal orchestration state:
  - `icloud_orchestration_runs`
  - `icloud_orchestration_batches`
- Added schema sync for the internal orchestration tables.
- Added internal orchestration service:
  - dry-run planning without download
  - start/run until cleanup review
  - explicit cleanup continuation
  - per-batch checkpointing
  - report writing
  - staged-unknown, partial-workspace, drop-zone, and active-operation preflight checks
- Added internal operator script:
  - `backend/scripts/run_icloud_internal_loop.py`
- Generalized the durable `ordinary_still_only` gate so a batch may contain more than one ordinary still logical item.
- Preserved strict ordinary-still rules:
  - each selected logical item must have exactly one selected resource
  - selected resource must be the primary original
  - Live Photo motion resources block
  - RAW/sidecar/multi-resource/ambiguous/unsupported selected items block
  - adjustment-metadata-only candidates remain allowed only when the actual downloadable selection is primary-only ordinary still
- Preserved Source Intake authority:
  - acquisition only publishes selected resources into managed iCloud staging
  - Source Intake remains the only vault/database ingestion authority
  - cleanup only removes local staging files after Source Intake evidence exists
- Preserved source-wide guarded cleanup, with the orchestration invariant that only one acquisition batch is staged at a time.
- Cleanup continuation requires the exact existing confirmation phrase:

```text
DELETE LOCAL STAGING COPIES
```

## Operator command shape

Dry run:

```powershell
$env:PYTHONPATH='backend'
& '.\.venv\Scripts\python.exe' '.\backend\scripts\run_icloud_internal_loop.py' `
  --dry-run `
  --source-id 66 `
  --batch-size 1 `
  --total-limit 2 `
  --scan-limit 25 `
  --ordinary-still-only
```

Start execution and pause at cleanup review:

```powershell
$env:PYTHONPATH='backend'
& '.\.venv\Scripts\python.exe' '.\backend\scripts\run_icloud_internal_loop.py' `
  --execute `
  --source-id 66 `
  --batch-size 1 `
  --total-limit 2 `
  --scan-limit 25 `
  --ordinary-still-only
```

Continue reviewed local cleanup:

```powershell
$env:PYTHONPATH='backend'
& '.\.venv\Scripts\python.exe' '.\backend\scripts\run_icloud_internal_loop.py' `
  --continue-cleanup `
  --orchestration-run-id <orchestration_run_id> `
  --cleanup-dry-run-id <cleanup_dry_run_id> `
  --confirm 'DELETE LOCAL STAGING COPIES'
```

If the total limit is not yet reached, a successful cleanup continuation verifies staging is clean and proceeds to the next acquisition batch, pausing again at cleanup review.
