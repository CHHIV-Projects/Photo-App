# Milestone 12.62.12 — Cleanup / Reacquire / Non-Repeat Validation Loop

## Goal

Validate the production-like iCloud ingestion loop after guarded local staging cleanup:

```text
Acquire from iCloud
→ Source Intake
→ Cleanup Dry Run
→ Verified Cleanup Execution
→ Reacquire using non-repeat / skip-known mode
→ Confirm previously ingested and cleaned assets are not redownloaded unnecessarily
```

This milestone should prove that verified cleanup does not cause the system to repeatedly redownload already-ingested iCloud assets.

This is primarily a validation and integration milestone. Do not reinvent the existing skip-known/non-repeat acquisition logic. Inspect and reuse the existing implementation from the earlier Admin iCloud acquisition work where appropriate.

## Background

Milestone 12.62.11B implemented guarded cleanup execution for verified local iCloud staging files.

The next question is whether the full real-world loop behaves correctly after cleanup.

The user notes that skip-known / non-repeat acquisition logic already exists from earlier Admin-tab iCloud ingestion work. This milestone should inspect and leverage that existing logic rather than designing a new non-repeat system from scratch.

The long-term iCloud goal remains:

```text
One coordinated iCloud intake action:
Acquire from iCloud
→ avoid repeats
→ ingest through Source Intake
→ verify imported files
→ clean up local staging
→ report result / next safe action
```

This milestone does not implement one-click orchestration. It validates the pieces needed before that future consolidation.

## Safety-Critical Rules

This milestone touches acquisition, Source Intake, and cleanup sequencing. Preserve these rules:

- Acquisition may only download files into the selected Source Profile’s managed iCloud staging path.

- Acquisition must never write directly to Vault.

- Acquisition must never write directly to DB asset records or provenance.

- Source Intake remains the only path into Drop Zone, Vault, DB asset records, and provenance.

- Cleanup must only act on verified local staging files through the guarded 12.62.11B cleanup contract.

- Cleanup must never delete iCloud cloud-library data.

- Cleanup must never call any iCloud/cloud delete API.

- Cleanup must never delete Vault files.

- Cleanup must never delete Drop Zone files.

- Cleanup must never delete DB records.

- Cleanup must never delete provenance records.

- Cleanup must never delete Source Profile or source registry records.

- Reacquisition validation must not weaken non-repeat safety.

- Frontend status or buttons are not safety controls; backend must enforce operation guardrails.

- The app must not store Apple passwords, 2FA codes, session cookies, auth tokens, or secrets.

- Do not log secrets or expose credentials in reports.

### 3. Validate the full loop

