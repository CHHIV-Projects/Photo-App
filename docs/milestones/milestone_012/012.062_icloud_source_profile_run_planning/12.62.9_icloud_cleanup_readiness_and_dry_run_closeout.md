## 10. Post-Validation Addendum (2026-06-18)

Additional field findings from interactive validation after initial closeout:

- The observed `storage/exports/icloud/Chuck_iCloud` structure with date folders and no files is expected from prior cleanup history, not from the 12.62.9 dry-run UI changes.
- This was traced to a historical non-dry cleanup run that deleted staged files for the Chuck iCloud source.

Evidence timeline:

- Acquisition report recorded files downloaded to `storage/exports/icloud/Chuck_iCloud` (`downloaded_count: 108`, `total_files: 108`).
- Subsequent cleanup report for the same source recorded `dry_run: false` and `deleted_count: 108`.
- Current-day cleanup run from Ingestion recorded `dry_run: true` and `eligible_count: 0`, consistent with an already-cleared staging folder.

Operational interpretation:

- Empty date-based directories under iCloud staging can be a normal post-cleanup artifact when files have already been removed.
- Ingestion dry-run action in 12.62.9 cannot perform deletions by design and therefore cannot be the cause of missing staged files.

Code-level safety confirmation:

- Ingestion uses a helper that always submits cleanup requests with `dry_run=true`.
- Backend cleanup execution deletes files only when `dry_run` is false.

Operator guidance added:

- When staging appears empty unexpectedly, verify prior cleanup report history first (`storage/logs/icloud_cleanup_reports`) before attributing to current milestone UI behavior.
