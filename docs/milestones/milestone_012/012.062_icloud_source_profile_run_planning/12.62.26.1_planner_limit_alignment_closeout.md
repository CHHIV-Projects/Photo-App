# Milestone 12.62.26.1 - Planner Limit Alignment Closeout

## Summary

Aligned the iCloud new-count planner limits with the 12.62.26 Recent Sync 1000 cap.

The planner constants were active enforcement points because `exact_selection_adapter.py` and
`internal_loop_orchestrator.py` import `MAX_CANDIDATE_SCAN_LIMIT` and `MAX_TARGET_NEW_ITEM_COUNT`
from `new_count_planner.py`. Leaving those values at 500 would have allowed the protocol/schema/UI
to advertise 1000 while the planner rejected 1000-item runs.

## Files Changed

- `backend/app/services/icloud_acquisition/new_count_planner.py`
- `backend/tests/test_icloud_new_count_planner.py`
- `docs/milestones/milestone_012/012.062_icloud_source_profile_run_planning/12.62.26.1_planner_limit_alignment_closeout.md`

## Implementation Details

- Raised `DEFAULT_CANDIDATE_SCAN_LIMIT` from 500 to 1000.
- Raised `MAX_CANDIDATE_SCAN_LIMIT` from 500 to 1000.
- Raised `MAX_TARGET_NEW_ITEM_COUNT` from 500 to 1000.
- Added focused planner tests proving:
  - planner constants align to 1000,
  - grouped planner accepts `target_new_item_count=1000` and `candidate_scan_limit=1000`,
  - explicit adapter planner accepts `target_new_item_count=1000` and `candidate_scan_limit=1000`,
  - planner rejects values above 1000.
