# Milestone 12.62.18 — Bounded Internal iCloud End-to-End Loop Validation

## Recommended Reasoning Level

Extra High.

This milestone validates the complete internal iCloud exact-selection chain across acquisition, durable batch state, Source Intake handoff, provenance evidence, cleanup readiness, and guarded cleanup. It is safety-sensitive because it touches real iCloud acquisition, local staging, Drop Zone staging, Vault writes, provenance, and local staging cleanup.

## Goal

Perform a bounded real-account internal end-to-end validation of the new durable iCloud exact-selection path.

The target validation flow is:

```text
one safe exact-selection iCloud acquisition batch
→ durable acquisition run/batch/item/resource rows
→ internal batch Source Intake handoff
→ Vault / Asset / provenance evidence
→ batch marked ready for cleanup dry run
→ guarded cleanup dry run
→ guarded cleanup execution only after operator review
→ final evidence that staging is clean and Vault/DB/provenance remain intact
```

This milestone should prove the internal acquire → Source Intake → cleanup handoff works for one small batch before we design multi-batch automation.
