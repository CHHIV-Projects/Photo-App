# Milestone 12.62.26 — Raise Candidate Scan and Acquire Limits for Recent Sync

## Objective

Finalize the **Recent Sync** limit behavior for the current internal/admin iCloud single-flow routine.

This milestone is intentionally narrow.

Implement the limit changes needed so the existing iCloud recent-sync path can:

```text
scan up to 1000 recent iCloud candidates
acquire up to 500 logical assets by default
allow up to 1000 logical assets as an admin/large-run override
```

Do **not** implement historical backfill in this milestone.

## Product-owner decisions locked for this milestone

### Recent Sync

```text
candidate_search_cap default: 1000
total_limit default: 500 logical assets
admin/request max total_limit: 1000 logical assets
```

### Counting rule

Limits must count **logical assets**, not raw files:

```text
ordinary still = 1 logical asset
video = 1 logical asset
Live Photo HEIC + MOV pair = 1 logical asset, 2 resources/files
```

12.62.25 indicates this is already implemented correctly. Preserve it.

### Recent Sync purpose

Recent Sync is for normal ongoing use after onboarding, including high-volume cases such as a vacation where the user may take 500+ photos/videos.

The recent path should inspect enough newest candidates to find those new assets:

```text
scan newest 1000 candidates
select/acquire up to requested total_limit
default total_limit 500
max total_limit 1000
```
